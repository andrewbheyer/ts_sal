Detailed Description

QueryCondition objects are specialized ReadCondition objects that allow the application to also specify a filter on the locally available data.

The query (queryExpression) is similar to an SQL WHERE clause can be parameterized by arguments that are dynamically changeable by the setQueryParameters(List) operation.

This feature is optional. In the cases where it is not supported, the DataReader#createQueryCondition(String, List) will return null.

The triggerValue of a QueryCondition is like that of a ReadCondition with the additional condition that the data associated with at least one sample must be such that the queryExpression evaluates to true.

Parameters:
<TYPE>The concrete type of the data that can be read using the the DataReader that created this QueryCondition.
