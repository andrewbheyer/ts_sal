Creates a new DomainParticipant. The newly created DomainParticipant which will join the domain identified by domainId, with the desired DomainParticipantQos and attaches the optionally specified DomainParticipantListener to it.

int DDS.DomainParticipantFactoryOperations.delete_participant ( DDS.DomainParticipant  a_participant)

Deletes a DomainParticipant. This method deletes a DomainParticipant. A DomainParticipant cannot be deleted when it has any attached Entity objects. When the operation is called on a DomainParticipant with existing Entity objects, the operation returns RETCODE_PRECONDITION_NOT_MET.

Parameters:

Returns:
Possible return codes of the operation are: RETCODE_OK, RETCODE_BAD_PARAMETER or RETCODE_PRECONDITION_NOT_MET.

int DDS.DomainParticipantFactoryOperations.set_default_participant_qos ( DDS.DomainParticipantQos  qos)

Sets the default DomainParticipantQos of the DomainParticipant. This method sets the default DomainParticipantQos of the DomainParticipant which is used for newly created DomainParticipant objects, in case the no qos is specified.

The default DomainParticipantQos is only used when null is supplied as qos parameter in the create_participant method.

The DomainParticipant QosPolicy settings are always self consistent, because the they do not depend on each other. This means this operation never returns the RETCODE_INCONSISTENT_POLICY. The values set by this operation are returned by get_default_participant_qos(DomainParticipantQosHolder).

Parameters:
qosnew default qos.
Returns:
return code.
