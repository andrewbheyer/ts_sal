Detailed Description

[optional] Specifies whether it is allowed for multiple org.omg.dds.pub.DataWriters to write the same instance of the data and if so, how these modifications should be arbitrated.

Member Enumeration Documentation

enum org::omg::dds::core::policy::Ownership::Kind
Enumerator:
SHARED 

Indicates shared ownership for each instance. Multiple writers are allowed to update the same instance and all the updates are made available to the readers. In other words there is no concept of an "owner" for the instances. This is the default behavior.

EXCLUSIVE 

Indicates each instance can only be owned by one org.omg.dds.pub.DataWriter, but the owner of an instance can change dynamically. The selection of the owner is controlled by the setting of the OwnershipStrength. The owner is always set to be the highest-strength DataWriter object among the ones currently "active" (as determined by the Liveliness).
